# 一、核心思路：
vs code+git做总版本管理，并推送github
obsidian+zotero做文本知识流整理
其它专业软件做分项，最终导出文件为.zip或.PDF文件

# 二、配置流程

>[!info] 第一步：建立总库
>建立一个总库，名为Managed Object，这个总库是和Github联动的库，也就是说，他的目的是和Github联动，所有的内容可以现在本地编辑，然后放到总库中来。
>
>==**总库配置**==:
>	1、在里面建立相对应的Knowledge、Tech-Stach、Engineering、Research四个专属文件夹
>	2、在Obsidian里面载入Managed Object做为库，在VS Code里面同样载入Managed Object做为库，以后的文件修改全部集中于这个库中，他是本地与云端Github之间的数据库
>	

>[!info] 第二步：配置Git
>每次为了使得Github保持同步更新，需要通过VS code对任意更改进行push，配置git也是使得对应仓库就是四个子仓库
>
>==**Git配置**==:
>	1、在 VS Code 中打开总目录： 也就是打开 D:\Managed Object
>	2、打开终端：按 `Ctrl + ~` 唤起下方终端
>	3、进入子目录：在终端输入：cd 1.Knowledge
>	4、输入：git init
>	5、输入：git branch -M main
>	6、输入：git remote add origin https://github.com/你的名字/Knowledge.git 进行仓库关联	
>	7、递交初次版本：git add .
>	git commit -m "Initial commit for Knowledge"
>	git push -f origin main
>	8、对每一个子文件夹重复上述配置。- 输入 `cd ..\`，然后按一下键盘上的 **`Tab`** 键，终端会自动给你轮换显示当前目录下的文件夹名字（比如跳出 `1.Knowledge`），再按一下 `Tab`，就会变成 `2.Tech-Stack`。
>

>[!info] 第三步：配置云桥
>上述步骤中，只能用来同步代码和md文件，我们的目标是不仅同步代码和md文件，还要同步.zip与.PDF文件，因为这两个文件通常很大，所以我们采用坚果云搭建云桥的方式进行同步
>==搭建云桥步骤==
>	1、下载坚果云账号，并对其进行桌面常驻
>	2、在D盘建立“E”文件夹，表示电子文件，并在里面建立相应的电子文档，对这个文档进行同步，这个文档就会被坚果云建立一个影子文档，在C盘，这个时候，就可以把D盘里的这个文件删掉了
>	3、如果有需要上传的DF或者.ZIP文件，直接相当于附件，放到这个影子文档里面来，然后右键，释放电脑内存，使得电脑本地只保留影子文件，真正的真身在云桥里面
>	4、搭建“虫洞”，将这个文件夹引入到Managed Object对应的文件夹中来。配置虫洞：命令行输入mklink /J "D:\Managed Object\2.Tech-Stack\e-tech" "C:\Users\86158\Nutstore\1\E\2.E-TECH"
>	5、对每一个内容，都写一个.md文档，引入链接到文件中来就可以，就可以直接下载了
>	




# 三、工作流程
1、如果需要对某个.md文档进行更改，在obsidian里面进行修改，在vs code里面进行同步
2、如果需要对某个代码进行梗概，在vs code里面进行同步
3、PDF或者.ZIP文档只能是做为某个.md文档的附件，利用分享链接进行同步
